Pagination constructs to support cursor based pagination in the API

```graphql
input Pagination {
  first: Int
  after: String
  last: Int
  before: String
}
```


### Fields

#### [<code style={{ fontWeight: 'normal' }}>Pagination.<b>first</b></code>](#)<Bullet />[`Int`](/docs/testnet/graphql/scalars/int) <Badge class="secondary" text="scalar"/>
> The number of items to fetch in the next page traversing forward through the connection
> 

#### [<code style={{ fontWeight: 'normal' }}>Pagination.<b>after</b></code>](#)<Bullet />[`String`](/docs/testnet/graphql/scalars/string) <Badge class="secondary" text="scalar"/>
> The cursor to start fetching items after. If empty, data will be fetched from the beginning of the connection
> 

#### [<code style={{ fontWeight: 'normal' }}>Pagination.<b>last</b></code>](#)<Bullet />[`Int`](/docs/testnet/graphql/scalars/int) <Badge class="secondary" text="scalar"/>
> The number of items to fetch in the next page traversing backward through the connection
> 

#### [<code style={{ fontWeight: 'normal' }}>Pagination.<b>before</b></code>](#)<Bullet />[`String`](/docs/testnet/graphql/scalars/string) <Badge class="secondary" text="scalar"/>
> The cursor to start fetching items before. If empty data will be fetched from the end of the connection
>
