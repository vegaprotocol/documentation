An order update in Vega, if active it will be on the order book for the market

```graphql
type OrderUpdate {
  id: ID!
  price: String!
  timeInForce: OrderTimeInForce!
  side: Side!
  marketId: ID!
  size: String!
  remaining: String!
  partyId: ID!
  createdAt: String!
  expiresAt: String
  status: OrderStatus!
  reference: String!
  type: OrderType
  rejectionReason: OrderRejectionReason
  version: String!
  updatedAt: String
  peggedOrder: PeggedOrder
  liquidityProvisionId: ID
}
```


### Fields

#### [`id`](#) ([`ID!`](/docs/testnet/graphql/scalars/id))

Hash of the order data


#### [`price`](#) ([`String!`](/docs/testnet/graphql/scalars/string))

The worst price the order will trade at (e.g. buy for price or less, sell for price or more) (uint64)


#### [`timeInForce`](#) ([`OrderTimeInForce!`](/docs/testnet/graphql/enums/order-time-in-force))

The timeInForce of order (determines how and if it executes, and whether it persists on the book)


#### [`side`](#) ([`Side!`](/docs/testnet/graphql/enums/side))

Whether the order is to buy or sell


#### [`marketId`](#) ([`ID!`](/docs/testnet/graphql/scalars/id))

The market the order is trading on (probably stored internally as a hash of the market details)


#### [`size`](#) ([`String!`](/docs/testnet/graphql/scalars/string))

Total number of units that may be bought or sold (immutable) (uint64)


#### [`remaining`](#) ([`String!`](/docs/testnet/graphql/scalars/string))

Number of units remaining of the total that have not yet been bought or sold (uint64)


#### [`partyId`](#) ([`ID!`](/docs/testnet/graphql/scalars/id))

The party that placed the order (probably stored internally as the party's public key)


#### [`createdAt`](#) ([`String!`](/docs/testnet/graphql/scalars/string))

RFC3339Nano formatted date and time for when the order was created (timestamp)


#### [`expiresAt`](#) ([`String`](/docs/testnet/graphql/scalars/string))

Expiration time of this order (ISO-8601 RFC3339+Nano formatted date)


#### [`status`](#) ([`OrderStatus!`](/docs/testnet/graphql/enums/order-status))

The status of an order, for example 'Active'


#### [`reference`](#) ([`String!`](/docs/testnet/graphql/scalars/string))

The external reference (if available) for the order


#### [`type`](#) ([`OrderType`](/docs/testnet/graphql/enums/order-type))

The order type


#### [`rejectionReason`](#) ([`OrderRejectionReason`](/docs/testnet/graphql/enums/order-rejection-reason))

Why the order was rejected


#### [`version`](#) ([`String!`](/docs/testnet/graphql/scalars/string))

Version of this order, counts the number of amends


#### [`updatedAt`](#) ([`String`](/docs/testnet/graphql/scalars/string))

RFC3339Nano time the order was altered


#### [`peggedOrder`](#) ([`PeggedOrder`](/docs/testnet/graphql/objects/pegged-order))

PeggedOrder contains the details about a pegged order


#### [`liquidityProvisionId`](#) ([`ID`](/docs/testnet/graphql/scalars/id))

The liquidity provision this order was created from
